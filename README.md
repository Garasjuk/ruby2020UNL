# SOLID

#1. Каждый класс должен решать одну задачу

#2. Классы, функции, модули - должны быть открыты для расширения но закрыты для изменения

#3. Поедение наследующих класов не должно противоречить поведению базового класса

#4. Разделение интерфесов на более мелкие, методы имели те инерфейсы которые им необходимы

#5. Модули верхнего уровня не должны завилисть от модулей нижнего уровня. Все модули зависят от абстракции. Абстракция не зависит от деталей, детали зависят от абстракции.

# OOP

Наследование - передача структуры одного класса другому, другой класс в свою очередь может создавать дополнителные свойства которые не доступны классу от которого наследуется

Инкапсуляция - доступ к свойством и методам объекта происходить через специльные интерфейсы (геттеры и сетеры)

Полиморфизм - переопределение унаследованных свойств и методов, Свойства методы будут иметь однаковую спецификацию но различную реализцию

# PATTERN

#Строитель — это порождающий паттерн проектирования, который позволяет создавать сложные объекты пошагово. Строитель даёт возможность использовать один и тот же код строительства для получения разных представлений объектов.

#Декоратор — это структурный паттерн, который позволяет добавлять объектам новые поведения на лету, помещая их в объекты-обёртки.

#Посетитель — это поведенческий паттерн, который позволяет добавить новую операцию для целой иерархии классов, не изменяя код этих классов.
